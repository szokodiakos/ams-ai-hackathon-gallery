# CLAUDE.md

This is the AMS AI Hackathon Gallery — a showcase of games built during the hackathon.

## Development Workflow

Development is spec-driven using OpenSpec.

1. Create a branch named `claude/<issue-number>-<short-description>`
2. Run `/opsx:propose` to create a change with proposal, design, and tasks
3. Run `/opsx:apply` to implement the tasks
4. Write tests for all new functionality
5. Commit the OpenSpec artifacts alongside the code
6. Create a PR with `Closes #<issue-number>` in the body

## Code Standards

- Use TypeScript with strict mode
- Write tests for all new functionality
- Keep functions small and focused
- Use descriptive variable and function names

## Commands

- `npm install` — install dependencies
- `npm test` — run tests (once test framework is set up)
- `npm run build` — build the project

## Project Structure

- `openspec/` — OpenSpec specs, changes, and configuration
- `.claude/` — Claude Code commands and skills (generated by OpenSpec)
- `src/` — Application source code
- `public/` — Static assets

## Rules

- NEVER push directly to the main branch. Always create a PR.
- NEVER merge your own PRs. The human reviewer will merge after approval.
- If you are unsure about a requirement, ASK. Do not guess.
- Keep PRs focused on a single issue.
- ALWAYS include `<!-- claude-watcher -->` at the end of every GitHub comment you post (issue comments, PR comments, review replies). This hidden marker lets the watcher distinguish your comments from the human's.
